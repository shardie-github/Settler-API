# Customer Journey Map: Developer → Scale Customer

**From first signup to enterprise scale—mapping the Settler customer journey.**

---

## Overview

This journey map tracks a typical customer from initial awareness through scale adoption, highlighting touchpoints, emotions, and opportunities for optimization.

**Persona:** DevOps/Backend Engineer at Mid-Market E-commerce Company  
**Goal:** Automate reconciliation between Shopify and Stripe  
**Timeline:** 0-12 months

---

## Stage 1: Awareness (Week 0)

### Touchpoints
- **Twitter/X** — Product announcement, developer community posts
- **Hacker News** — "Show HN" post
- **Indie Hackers** — Community discussion
- **Product Hunt** — Launch day
- **Developer Blogs** — Technical deep-dives

### Customer State
- **Emotion:** Curious, skeptical
- **Pain Point:** Spending 2-3 hours daily on manual reconciliation
- **Question:** "Is this real? Can it actually solve my problem?"

### Actions
- Reads blog post / sees social post
- Clicks through to website
- Skims homepage and pricing
- **Decision Point:** Sign up for free account or leave

### Optimization Opportunities
- **Clear value prop** on homepage
- **Social proof** (testimonials, usage stats)
- **Free tier** prominently displayed
- **Quick demo video** or playground link

---

## Stage 2: Signup & First Experience (Day 1)

### Touchpoints
- **Website** — Signup form
- **Email** — Welcome email (developer or CTO version)
- **Dashboard** — First login
- **API Key** — Generated automatically

### Customer State
- **Emotion:** Hopeful, slightly overwhelmed
- **Pain Point:** "How do I actually use this?"
- **Question:** "Where do I start?"

### Actions
- Signs up (no credit card required)
- Receives welcome email
- Logs into dashboard
- Sees API key and quick start guide
- **Decision Point:** Try integration or abandon

### Optimization Opportunities
- **Onboarding checklist** in dashboard
- **"Try Playground"** CTA (no code required)
- **Copy API key** button with one click
- **5-minute quickstart** video embedded
- **Example code** ready to copy/paste

---

## Stage 3: First Integration (Day 1-2)

### Touchpoints
- **Documentation** — Getting started guide
- **SDK** — npm install and first API call
- **Playground** — Testing without code
- **Discord** — Community support

### Customer State
- **Emotion:** Excited, nervous
- **Pain Point:** "Will this work with my setup?"
- **Question:** "Am I doing this right?"

### Actions
- Installs SDK (`npm install @settler/sdk`)
- Creates first reconciliation job
- Tests with sample data
- **Decision Point:** Continue or give up

### Optimization Opportunities
- **Step-by-step tutorial** with code examples
- **Playground** for visual testing
- **Discord support** (real-time help)
- **Error messages** that guide to solution
- **Success confirmation** when job created

---

## Stage 4: First Results (Day 2-3)

### Touchpoints
- **Dashboard** — First reconciliation report
- **Email** — "Your first reconciliation completed"
- **API** — Fetching report data
- **Webhooks** — Real-time updates (if configured)

### Customer State
- **Emotion:** Relieved, impressed (if it works)
- **Pain Point:** "Are these results accurate?"
- **Question:** "What do I do with unmatched transactions?"

### Actions
- Views first report
- Sees matched/unmatched transactions
- Reviews accuracy
- **Decision Point:** Trust results and proceed, or investigate further

### Optimization Opportunities
- **"First report" email** with explanation
- **Accuracy metrics** prominently displayed
- **Unmatched transactions** with suggested actions
- **Matching rule suggestions** based on results
- **Success story** ("You saved X hours this week")

---

## Stage 5: Production Rollout (Week 1-2)

### Touchpoints
- **Dashboard** — Configuring production jobs
- **Documentation** — Advanced features
- **Email** — Weekly check-in
- **Support** — Email/Discord for questions

### Customer State
- **Emotion:** Confident, focused
- **Pain Point:** "How do I scale this?"
- **Question:** "What else can Settler do?"

### Actions
- Configures scheduled jobs
- Sets up webhooks for real-time reconciliation
- Adds more platform connections
- Trains finance team on exception handling
- **Decision Point:** Expand usage or stay minimal

### Optimization Opportunities
- **"Scale up" email** with advanced features
- **Best practices guide** for production
- **Finance team onboarding** resources
- **Case studies** from similar companies
- **Feature discovery** (show unused features)

### Customer State

- **Question:** "What's the best way to handle edge cases?"

### Actions

- Implements custom matching logic

## Stage 7: Scale (Month 1-3)

### Touchpoints
- **Dashboard** — Usage metrics, billing
- **Email** — Usage reports, upgrade prompts
- **Support** — Scaling questions
- **Sales** — Upgrade conversations (if needed)

### Customer State
- **Emotion:** Committed, scaling
- **Pain Point:** "Am I on the right plan?"
- **Question:** "What happens when I hit limits?"

### Actions
- Monitors usage (approaching free tier limit)
- Reviews pricing tiers
- Upgrades to paid plan (if needed)
- Expands to more platforms
- **Decision Point:** Upgrade or optimize usage

### Optimization Opportunities
- **Usage alerts** before hitting limits
- **Upgrade prompts** with clear ROI
- **Plan comparison** tool
- **Success metrics** ("You've reconciled X transactions")
- **Referral program** (invite friends, get credits)

- **Pain Point:** "Do we need Enterprise?"

- Reviews compliance requirements

- **Customer success** team outreach

## Key Metrics by Stage

### Stage 1: Awareness
- **Conversion Rate:** 2-5% (visitor → signup)
- **Time to Signup:** <5 minutes

### Stage 2: Signup
- **Activation Rate:** 60-70% (signup → first API call)
- **Time to First API Call:** <24 hours

### Stage 3: First Integration
- **Success Rate:** 80-90% (first API call → job created)
- **Time to First Job:** <2 hours

### Stage 4: First Results
- **Accuracy:** 95-99% (first report accuracy)
- **Time to First Report:** <24 hours

### Stage 5: Production Rollout
- **Production Rate:** 50-60% (first job → production)
- **Time to Production:** 1-2 weeks

### Stage 6: Optimization
- **Accuracy Improvement:** +2-5% (after optimization)
- **Time to Optimize:** 1-2 weeks

### Stage 7: Scale
- **Upgrade Rate:** 30-40% (free → paid)
- **Time to Upgrade:** 1-3 months

### Stage 8: Power User
- **Feature Adoption:** 3-5 advanced features
- **Community Participation:** Active in Discord

### Stage 9: Enterprise Consideration
- **Enterprise Conversion:** 5-10% (Scale → Enterprise)
- **Sales Cycle:** 1-3 months

### Stage 10: Enterprise Customer
- **Retention Rate:** 95%+ (annual renewal)
- **Expansion Rate:** 20-30% (year-over-year growth)

## Success Criteria

**Healthy Customer Journey:**
- ✅ 60%+ activation rate (signup → first API call)
- ✅ 80%+ first job success rate
- ✅ 95%+ first report accuracy
- ✅ 50%+ production rollout rate
- ✅ 30%+ free → paid conversion
- ✅ 95%+ annual retention (paid customers)

**Unhealthy Signals:**
- ❌ <40% activation rate → Improve onboarding
- ❌ <70% first job success → Improve docs/support
- ❌ <90% accuracy → Improve matching defaults
- ❌ <30% production rollout → Improve production guide
- ❌ <20% free → paid → Improve upgrade prompts
